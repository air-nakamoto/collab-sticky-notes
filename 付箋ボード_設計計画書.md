# 📌 付箋ボード — 仕様設計計画書

## 1. プロダクト概要

### プロダクト名
**付箋ボード**（リアルタイム共有付箋ボード）

### コンセプト
複数メンバーがルームIDを共有するだけで、同じ付箋ボードをリアルタイムに操作できるWebアプリケーション。付箋の追加・移動・編集に加え、矢印・点線丸・点線四角の図形を自由に配置し、360度回転・サイズ変更・レイヤー制御が可能。盤面のPNG/HTML保存機能つき。

### ターゲットユーザー
- TRPGセッションのブレスト・プロット整理
- チームのアイデア出し・ふりかえり
- 少人数でのリアルタイムコラボレーション

### 技術スタック（現行）
- React（JSX、Hooks）
- CSS-in-JS（インラインスタイル）
- Google Fonts（Klee One, Zen Kurenaido）
- 永続化: `window.storage`（Claude Artifact用 shared persistent storage）
- 同期方式: 1.5秒間隔ポーリング

## 2. 画面構成

### 2-1. ジョインスクリーン（JoinScreen）

ルームへの入室画面。2ステップ構成。

**ステップ❶ 表示名入力**
- テキストフィールド（最大12文字）
- 必須。未入力でボタン押下時はエラーメッセージ表示

**ステップ❷ ボードを選ぶ**
- 「✨ 新しいボードを作成する」→ ルームID自動生成 → 即入室
- 「🔗 ルームIDで既存ボードに参加」→ ID入力欄が展開 → 存在チェック後に入室
- 使い方説明を常時表示

**バリデーション・エラー処理**
- 名前未入力: 「名前を入力してください」
- ルームID不足: 「ルームIDを正しく入力してください」
- ルーム不存在: 「ルーム「XXXXXX」が見つかりません」
- 保存失敗: 「作成に失敗しました」

### 2-2. ボードスクリーン（BoardScreen）

メイン操作画面。上部ツールバー＋無限スクロールキャンバス。

**ツールバー構成（左→右）:**

2. 🔗 ルームID（クリックでコピー → トースト通知）
3. メンバーアイコン列（プレゼンス表示）
4. セパレータ
5. 付箋テキスト入力フィールド
6. 付箋カラー選択（6色）
7. 「追加」ボタン
8. セパレータ
9. 🔷 図形トグルボタン
10. 💾 保存ドロップダウン
11. 枚数・図形数カウンタ
12. ❓ ヘルプ
13. 退室ボタン

**図形パネル（展開式サブツールバー）:**
- 図形種類ボタン: ➜矢印 / ◯点線丸 / ▭点線四角
- 図形カラー選択（8色）

## 3. データ構造

### 3-1. ルームID
- 6文字英数字（大文字、紛らわしい文字除外）
- 使用文字: `ABCDEFGHJKLMNPQRSTUVWXYZ23456789`

### 3-2. ストレージキー設計

```
board:{roomId}        → { notes, shapes, updatedAt }  (shared: true)
presence:{roomId}:{name} → { name, color, lastSeen }   (shared: true)
```

### 3-3. Note（付箋）オブジェクト

```javascript
{
  id: string,          // ユニークID（ランダム文字列+タイムスタンプ）
  text: string,        // 付箋テキスト
  x: number,           // X座標（px）
  y: number,           // Y座標（px）
  width: number,       // 幅（デフォルト: 195）
  color: string,       // カラー名: "yellow"|"pink"|"blue"|"green"|"orange"|"purple"
  rotation: number,    // 回転角度（-2.5〜+2.5、ランダム付与）
  zIndex: number,      // 重なり順
  author: string       // 作成者の表示名
}
```

### 3-4. Shape（図形）オブジェクト

```javascript
{
  id: string,          // ユニークID
  type: string,        // "arrow" | "dotted-circle" | "dotted-rect"
  x: number,           // X座標（px）
  y: number,           // Y座標（px）
  w: number,           // 幅（px）
  h: number,           // 高さ（px）
  rotation: number,    // 回転角度（0-359度）
  color: string,       // HEXカラー（例: "#e74c3c"）
  zIndex: number,      // 重なり順
  layer: string        // "above"（付箋の上）| "below"（付箋の下）
}
```

### 3-5. Presence（接続情報）オブジェクト

```javascript
{
  name: string,        // 表示名
  color: string,       // ユーザーカラー（名前のcharCodeから自動割り当て）
  lastSeen: number     // 最終更新タイムスタンプ（15秒超過で非表示）
}
```

## 4. 図形の仕様

### 4-1. 図形タイプ

| type          | 表示      | デフォルトサイズ  | 描画方法                          |
|---------------|-----------|-------------------|-----------------------------------|
| arrow         | ➜ 矢印    | 160×40px          | SVG line + marker（矢じり）       |
| dotted-circle | ◯ 点線丸  | 120×120px         | SVG ellipse, strokeDasharray="8 5"|
| dotted-rect   | ▭ 点線四角 | 160×100px         | SVG rect, rx=4, strokeDasharray   |

### 4-2. 図形コントロール

図形をクリックで選択すると以下のUIが表示される:

**上部: 回転ハンドル**
- 青い丸（↻）アイコン、図形の上辺中央から32px上に表示
- ドラッグで360度自由回転
- 下部に現在角度を表示（例: "45°"）

**右下: リサイズハンドル**
- 白い□（14×14px、青ボーダー）
- ドラッグで幅・高さを変更（最小: 30×20px）

**右側: 縦メニュー**（図形の右端+12pxの位置に表示）
- ⬆/⬇ レイヤー切替（前面/背面）
- カラーパレット（8色、丸ボタン）
- 🗑 削除

### 4-3. レイヤー制御

- `layer: "above"` → 付箋の上に描画（デフォルト）
- `layer: "below"` → 付箋の下に描画（背景として使用）
- 切替時に zIndex を調整: below=0, above=maxZ+1

### 4-4. 選択解除の制御

- 空白キャンバスクリック → 選択解除
- ドラッグ操作後 → `didDragRef`フラグにより選択を維持
- 付箋クリック → 図形選択を解除（`onDeselect`コールバック）
- 図形の右側メニュー操作 → `e.stopPropagation()`で選択維持

## 5. リアルタイム同期

### 5-1. 同期方式

**ポーリング（1.5秒間隔）** で共有ストレージを監視。

```
[ユーザー操作] → setNotes/setShapes → debounce(200ms) → saveBoard()
                                                            ↓
                                              shared storage更新
                                                            ↓
[他ユーザー] ← poll(1500ms) ← loadBoard() ← shared storage
```

### 5-2. 競合回避

- `lastSyncRef`: 自分が書き込んだ直後（800ms以内）はリモート更新をスキップ
- ドラッグ中/テキスト編集中: 操作中のアイテムだけローカル版を維持し、他はリモートで上書き
- 回転中/リサイズ中: 同様に操作中の図形はローカル版を優先

### 5-3. プレゼンス

- 各ユーザーがポーリングのたびに`presence:{roomId}:{name}`を更新
- `lastSeen`から15秒超過で表示から除外
- ツールバーにアバター（名前の頭文字＋ユーザーカラー）を表示

## 6. 盤面保存機能

### 6-1. PNG画像保存

Canvas 2D APIで盤面を直接描画してダウンロード。

**処理フロー:**
1. `document.createElement("canvas")`で描画用キャンバス作成
2. 背景色 `#f5f0e8` で塗りつぶし
3. 全図形を描画（translate + rotate で回転対応）
4. 全付箋を描画（角丸矩形 + 左ボーダー + テキスト折り返し）
5. `canvas.toBlob()` → `URL.createObjectURL` → ダウンロード

**注意点:**
- `ctx.roundRect()`は未サポートブラウザあり → `drawRoundedRect()`（quadraticCurveTo使用）で代替
- キャンバスサイズ上限: 4000×4000px

### 6-2. HTML保存

付箋と図形をインラインスタイルのHTML要素として書き出し、スタンドアローンHTMLとして保存。

**処理フロー:**
1. 図形 → `position:absolute` + インラインSVGの`<div>`
2. 付箋 → `position:absolute` + スタイル付き`<div>`
3. テキストはHTMLエスケープ（`<` → `&lt;`）
4. `Blob` → ダウンロード

## 9. 既知の制約・課題

### 9-1. 現行の制約

1. **ポーリング方式のため1〜2秒のタイムラグ**がある（WebSocketではない）
2. **同時編集の厳密な競合解決がない**（last-write-wins方式）
3. **window.storage依存**（Claude Artifact環境限定、外部デプロイには別のバックエンド必要）
4. **付箋のテキスト編集中にリモート更新が入ると内容が巻き戻る可能性**がある
5. **PNG保存はCanvas 2Dで再描画するため、フォントやレイアウトが完全一致しない**場合がある

### 9-2. 将来の改善候補

1. **Firebase Realtime Database / Supabase Realtime への移行** → 真のリアルタイム同期
2. **カーソル共有**（他ユーザーのカーソル位置をリアルタイム表示）
3. **Undo/Redo**（操作履歴の管理）
4. **付箋のリサイズ対応**
5. **テキスト付き図形**（ラベル付き矢印など）
6. **直線・フリーハンド描画**
7. **画像アップロード対応**（背景画像、スタンプ的な貼り付け）
8. **html2canvas or dom-to-image による高精度PNG出力**
9. **付箋のグルーピング・ロック機能**
10. **ルームのパスワード保護**

## 10. 外部デプロイ時の移行ガイド

現行の`window.storage`を外部バックエンドに置き換える場合の指針。

### 10-1. Firebase Realtime Database の場合

```
// 現行
window.storage.set("board:{roomId}", JSON.stringify(data), true)
window.storage.get("board:{roomId}", true)

// Firebase置き換え
import { ref, set, onValue } from "firebase/database";
set(ref(db, "rooms/" + roomId), data);
onValue(ref(db, "rooms/" + roomId), (snap) => { ... });
```

- ポーリング → `onValue`リスナーに置き換え（リアルタイム通知）
- プレゼンスは Firebase Realtime Database の `.info/connected` + `onDisconnect` で実装

### 10-2. Supabase Realtime の場合

```
// テーブル: boards (room_id TEXT, data JSONB, updated_at TIMESTAMPTZ)
supabase.from("boards").upsert({ room_id, data });
supabase.channel("room:" + roomId).on("postgres_changes", ..., callback);
```

### 10-3. Vercel + Next.js デプロイの場合

1. `pages/index.tsx` にコンポーネントを配置
2. Firebase/Supabaseの初期化を `lib/firebase.ts` に
3. 環境変数で API キーを管理
4. `vercel deploy` でデプロイ
